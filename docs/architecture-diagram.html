Start of every run · owns the recipe: cell · chroma_key · states (frames/fps/loop/action).
1 · PREPARE prepare_sprite_run.py
Generates a per-state layout-guide image + prompt text (and the empty raw/·frames/ skeleton).
2 · GENERATE kuma:image-gen (external)
One image per state = one horizontal strip. Never produces the whole atlas in a single generation.
3 · EXTRACT extract_sprite_row_frames.py
Chroma removal → connected components → group by x-center → fit_to_cell. If it can't reach the frame count it blocks — no silent fallback.
4 · FRAMES frames/<state>/frame-N.png
+ frames-manifest.json (per-frame QA: empty/sparse, edge bleed, chroma-adjacent, size outlier).
optional (baked at compose)
5 · CURATE serve_curation.py · optional
Webview select / reject + affine nudge (move·scale·rotate·shear). Result is curation.json, a non-destructive sidecar — source PNGs untouched, outside the write-lock.
6 · COMPOSE compose_sprite_atlas.py
Places each frame into a cell slot. Sheet = (max frames × cellW) × (states × cellH).
7 · OUTPUT sprite-sheet-alpha.png
Real alpha channel, no leftover chroma fringe — verified against white backgrounds.
SSoT manifest.json . frame_layout runtime SSoT
The absolute rects (x,y,w,h) + per-state fps/loop the engine consumes. Game code never recovers frames from alpha at runtime. QA: preview_animation.py → contact sheet + GIF.

🟩 Single cell model

One cell (square 256 default) drives generation · extraction · atlas geometry identically. No gen_cell / atlas_cell split — generate rect and the atlas is rect.

🟥 Concurrency lock

runio.py + .sprite-gen.lock — single writer per character folder. A second writer fails loudly with the holder's pid. curation.json sits outside the lock, so edit sessions never block a running compose.

⬜ inverse · import

unpack_atlas_run.py: finished atlas / loose PNG set → curator run dir (grid > manifest > auto). export_curated_pngs.py: bake curation transforms back into named PNGs.
