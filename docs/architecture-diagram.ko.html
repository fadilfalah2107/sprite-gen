sprite-gen 아키텍처 — 컴포넌트-행(row) 파이프라인

그림 한 장 넣으면 → 게임에 바로 쓰는 스프라이트 아틀라스가 나온다. 상태(state)별로 가로 스트립 한 줄씩 생성.
숫자/런타임 SSoT 일반 단계 외부 생성(image-gen) 선택 · 비파괴 핵심 개념
STAGE 0 정체성 게이트 · BLOCKING
base 이미지 → idle anchor 승인. 승인 후 base 는 행 입력에서 폐기 — anchor 가 정체성 truth 가 되고, 이후 모든 action 행은 모션만 소유한다.
정체성 (anchor 만 전달)
SSoT sprite-request.json 숫자 SSoT
모든 run 의 시작점 · 레시피 소유: cell · chroma_key · states(frames/fps/loop/action).
1 · 준비 prepare_sprite_run.py
상태별 layout-guide 이미지 + prompt 텍스트 생성 (빈 raw/·frames/ 골격도 함께).
2 · 생성 kuma:image-gen (외부)
상태당 이미지 1장 = 가로 스트립. 아틀라스를 통째로 한 번에 만들지 않는다.
3 · 추출 extract_sprite_row_frames.py
크로마 제거 → connected components(덩어리 탐지) → x중심으로 묶기 → fit_to_cell. 프레임 수를 못 채우면 block — silent fallback 없음.
4 · 프레임 frames/<state>/frame-N.png
+ frames-manifest.json (프레임별 QA: 빈/엣지 누출/크로마 잔여/크기 이상).
선택 (있으면 compose 때 baked)
5 · 큐레이션 serve_curation.py · 선택
웹뷰에서 프레임 선택/거부 + affine 보정(이동·스케일·회전·shear). 결과는 curation.json 비파괴 사이드카 — 원본 PNG 안 건드림, 쓰기-락 밖.
6 · 합성 compose_sprite_atlas.py
각 프레임을 cell 슬롯에 배치. 시트 = (최대 프레임수 × cell너비) × (상태수 × cell높이).
7 · 산출물 sprite-sheet-alpha.png
진짜 알파 채널, 크로마 잔여 없음 — 흰 배경 대비 검증됨.
SSoT manifest.json . frame_layout 런타임 SSoT
게임 엔진이 소비하는 절대 좌표 rect(x,y,w,h) + 상태별 fps/loop. 런타임에 알파로 프레임을 추정하지 않는다. QA: preview_animation.py → contact 시트 + GIF.

🟩 단일 cell 모델

cell 하나(기본 square 256)가 생성·추출·아틀라스 기하 3단계를 똑같이 지배한다. gen_cell / atlas_cell 분리 없음 — rect 로 생성하면 아틀라스도 rect.

🟥 동시성 락

runio.py + .sprite-gen.lock — 캐릭터 폴더당 단일 writer. 두 번째 writer 는 holder pid 와 함께 시끄럽게 실패. curation.json 은 락 밖이라 편집 세션이 compose 를 막지 않음.

⬜ inverse · import

unpack_atlas_run.py: 완성 아틀라스 / PNG 세트 → 큐레이터 run 디렉터리 (grid > manifest > auto). export_curated_pngs.py: 큐레이션 보정을 named PNG 로 baked.
